Для чего это нужно

InvisibleDetector создан чтобы сделать работу с LLM-текстами прозрачной. Для этого пользователь должен знать что содержится в тексте — даже если это невидимо глазу. Инструмент полностью бесплатный и работает без сервера.

Зачем нужен детектор?

Почему в текстах от LLM появляются скрытые символы

Языковые модели (ChatGPT, Claude, Gemini и др.) иногда включают в ответы невидимые символы Unicode — нулевые пробелы, мягкие переносы, соединители. Эти символы не видны на экране, но присутствуют в тексте: они могут ломать поиск, нарушать парсинг, служить для скрытой пометки контента или просто засорять данные.

💡 Пример: строка «привет» может содержать U+200B между буквами и выглядеть нормально, но при сравнении с «привет» — не совпадёт.

Как работает Декодер

1

Считать текст

Текст из textarea считывается как JavaScript-строка. Каждый символ в JS — это кодовая единица UTF-16.

2

Сравнить каждый символ с таблицей

Перебираем символы через for...of. Сравниваем с 19 записями в таблице невидимых символов.

for (const ch of text) {
  const sym = TABLE.find(s => s.char === ch);
  if (sym) found[sym.code] = (found[sym.code] || 0) + 1;
}
3

Вывод результата

Формируем текст без невидимых символов и статистику: сколько символов всего, сколько невидимых, каких типов и сколько раз каждый встретился.
