Общие вопросы

Это символы, которые не отображаются визуально в тексте, но занимают место в кодировке.

Примеры: U+200B (нулевой пробел), U+FEFF (метка порядка байт), U+00AD (мягкий перенос), U+E0100 (вариантный селектор 17). Они есть в строке, но их не увидеть.

Это блок специальных символов-модификаторов.

Вариантный селектор всегда идёт после базового символа, используется для его альтернативного варианта отображения.

Точная причина зависит от модели. Чаще всего это происходит из-за того, что модель обучалась на текстах, содержащих эти символы, теперь же она воспроизводит их. Иногда это может быть намеренная «водяная метка» для отслеживания источника текста.

Особенно часто встречается U+2014 (длинное тире) вместо обычного дефиса — это типографическая особенность, а не техническая проблема.

Для обычного чтения — ничем. Проблемы возникают, когда текст используется программно:

1. Сравнение строк даёт неожиданный результат ("привет" ≠ "привет")

2. Поиск не находит слово

3. CSV или JSON парсится с ошибкой

4. Код не работает, если символ попал в имя переменной

Про инструмент

Нет. Весь анализ происходит прямо в вашем браузере на JavaScript. Текст никуда не передаётся — нет сервера, нет базы данных, нет логов. Вы можете отключить интернет после загрузки страницы и инструмент продолжит работать.

До 10 МБ. На очень больших текстах (10+ МБ) может быть задержка.

Генератор добавляет невидимые символы рядом с обычными. Вероятность вставки задаётся ползунком «Интенсивность» (например, при значении 3 — 30%), длина вставки определяется случайно от 1 до 5 символов.

Используется для тестирования детектора.

Для разработчиков

Подключите файл detector.js и используйте глобальный объект window.Detector:

const result = Detector.analyzeText(myText);
const clean = Detector.removeInvisible(myText);
const html = Detector.highlightInvisible(myText);
const encoded_text = Detector.encode(mySymbol, myText);
const decoded_text = Detector.decode(myText);

Откройте js/detector.js и добавьте запись в массив INVISIBLE_CHARS:

{ code: 'U+XXXX', name: 'Название', char: '\uXXXX', severity: 'high' }

Значения severity: highest, high, medium, low
