Общие вопросы

Это символы, которые не имеют визуального представления — они не занимают место на экране и не печатаются. Примеры: U+200B (нулевой пробел), U+FEFF (метка порядка байт), U+00AD (мягкий перенос). Они есть в строке, но глазом их не увидеть.

Точная причина зависит от модели. Чаще всего это происходит из-за особенностей токенизации — модель обучалась на текстах, содержащих эти символы, и воспроизводит их. Иногда это может быть намеренная «водяная метка» для отслеживания источника текста.

Особенно часто встречается U+2014 (длинное тире) вместо обычного дефиса — это типографическая особенность, а не техническая проблема.

Для обычного чтения — ничем. Проблемы возникают когда текст используется программно:

• Сравнение строк даёт неожиданный результат ("привет" !== "при­вет")

• Поиск не находит слово

• CSV или JSON парсится с ошибкой

• Код не работает если символ попал в имя переменной

Про инструмент

Нет. Весь анализ происходит прямо в вашем браузере на JavaScript. Текст никуда не передаётся — нет сервера, нет базы данных, нет логов. Вы можете отключить интернет после загрузки страницы и инструмент продолжит работать.

Практически без ограничений — браузер справится с текстами в несколько мегабайт. На очень больших текстах (10+ МБ) может быть заметная задержка из-за построения HTML-подсветки.

Генератор случайно вставляет невидимые символы в текст после случайных символов. Ползунок «Интенсивность» задаёт вероятность вставки: при значении 3 каждый символ с вероятностью 3% получит невидимый «сосед».

Используется для тестирования детектора или для обфускации контента.

Для разработчиков

Подключите файл detector.js и используйте глобальный объект window.Detector:

const result = Detector.analyzeText(myText);
const clean  = Detector.removeInvisible(myText);
const html   = Detector.highlightInvisible(myText);

Откройте js/detector.js и добавьте запись в массив INVISIBLE_CHARS:

{ code: 'U+XXXX', name: 'Название', char: '\uXXXX', severity: 'high' }

Значения severity: high, medium, low
